我的技术路线与学佛之路（三）

Date Mon 12 March 2012 By renewjoy Category 技术路线与学佛之路. Tags python 哲学 黑客

在接触python之前，我觉得我应该会献身于C/C++。

因为C是我接触的第一门编程语言，他让我了解了计算机可以实现人的想法，并且按人的意志去完成一些很麻烦的事情。现在想来，我倒是用它来模拟和测试一些数学上的趣味东西，但貌似我没有真正的用C写个像样的程序出来。

C语言很麻烦，但这倒让我们更多的了解了计算机程序的内部原理。后来，我曾一度的觉得学校应该先开python作为入门级语言，而接下来才开C语言，因为初学者是很难一下子接触那么多的东西的。

我接触了很多人，也了解了他们学习C语言的痛苦，因它需要处理太多的细节了。尤其是女生，我甚至认为，搞IT的女生，学Python比学C语言价值大得多，至少可以常用于工作中。

如果你觉得我是在说Python语言比C语言好，那你也许错了。想了解编程语言的区别，网上有太多的文章来讲这个，我个人比较喜欢《黑客与画家》作者的观点。推荐你去看看这本书，也是友人推荐我看的。

也对，我是喜欢Python，非常强烈，但这并不代表你也要这样地喜欢。多年以后，我问自己，为什么喜欢Python呢？可能因为我自己编程世界的哲学观吧。

嗯嗯，我又跑题了，跑到哲学上去了。是的，我开始接触了哲学。也仅仅是接触而已，因为我并不是很熟悉这个内容。如果问因缘，那仅仅是因为我看过《Unix程序设计哲学》和最近才看的《中国哲学简史》（如果你想了解“三教九流”的来源，推荐看看这本由国学大师冯友兰用英文写成而后由他学生翻译过来的书）。

哲学是对人生、世界、宇宙的系统性思考。这样看来，我多年前就已经在接触哲学了，因为佛所说的话，就是哲学。

有人问我，为什么要接触并了解哲学？我说，因为他对人生有指导性的作用，对编程开发也有大大的益处。这便是我上面提到的那本书了，程序设计的哲学。

近几年很火的敏捷开发，就是很好的一种开发哲学。我很喜欢的几句话:

Make it run.
Make it right.
Make it fast.
Make it small.

我想，以后如果遇到实在不能接受佛学的人，我会说，我在学习哲学。

很多人不明白，为什么搞开源的人最后会信佛，甚至认为开源怎么和佛法扯上关系了呢？

我不清楚自由软件教主Richard Stallman的信仰是不是佛，但他没有信仰是不可能有今天的成就的，或许他不信仰佛法，但我认为他的信仰的思想应该和佛学的思想很相近。

我不确定搞开源的人最后都会信佛，但我感觉信佛的IT人最后都会或多或少地从事开源相关的工作。

我必然（一切的偶然都是必然）地接触了Python，这真是猛蛇过江啊。
